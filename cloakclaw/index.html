Why CloakClaw?

🧠

Two-Pass Detection

Regex patterns catch structured data. Local Ollama LLM catches names, companies, and context-dependent entities that regex misses.

🏠

100% Local

Cloaking runs entirely on your machine. No cloud calls. No API keys needed for the privacy layer itself.

🔐

AES-256 Encrypted

Entity mappings stored in encrypted SQLite. Optional password protection. Auto-expiry after 7 days.

🔄

Full Round-Trip

Cloak before sending, decloak when the response comes back. Originals are perfectly restored.

📄

6 Document Profiles

General, Legal, Financial, Email, Code, Medical — each profile targets the entity types that matter most.

🚫

Zero Telemetry

Nothing phones home. No analytics. No tracking. No data collection. Ever.

How It Works

1

Extract

PDF, DOCX, TXT, CSV, JSON — text extraction via poppler with pdfjs-dist fallback.

2

Detect

Two-pass NER: fast regex patterns, then local Ollama LLM for contextual analysis.

3

Replace

Generate realistic fakes — dollars scale proportionally, dates shift consistently, names get plausible alternatives.

4

Store

Encrypted SQLite mapping database. AES-256-GCM. Auto-expires after 7 days.

5

Decloak

When the LLM responds, reverse all substitutions. Originals perfectly restored.
